4. 모듈별 상세 구현 가이드라인

4.1. 모듈 1: 지능형 오디오 분할 (Audio Splitter)

* 기능: 10:49:40 분량의 m4a 파일을 1시간 단위로 분할하되, 분할 지점마다 앞뒤로 10초의 중첩 구간(Overlap)을 삽입함.
* 산출물: 분할된 개별 오디오 파일들 (예: part01.m4a, part02.m4a 등) 및 분할 지점 정보가 담긴 로그 파일.
* 확인 절차: 콘솔에 현재 분할 중인 파일명과 진행률을 표시하고, 작업 완료 후 폴더 내 생성된 파일 개수와 용량을 사용자가 직접 확인.

4.2. 모듈 2: WhisperX 텍스트 추출 및 강제 정렬 (STT & Alignment)

* 모델 설정: WhisperX large-v3 모델 사용, 정확도 우선 설정.
* 기능: 분할된 각 오디오 파일에 대해 JSON 형태의 워드 단위 타임스탬프를 추출하고, 음소 단위 강제 정렬(Forced Alignment)을 통해 싱크를 교정함.
* 산출물: 각 파트별 JSON 및 SRT 파일.
* 확인 절차: 실시간 CLI 로그를 통해 현재 처리 중인 문장과 배치(Batch) 진행 상황을 출력. 각 파일 작업 완료 시 사용자가 JSON/SRT 파일을 열어 인식률을 검토한 뒤 다음 파트 혹은 다음 단계 진행 여부를 결정.

4.3. 모듈 3: 자막 병합 및 오프셋 최적화 (Sub-Merger)

* 기능: 중첩 구간으로 인해 발생하는 타임스탬프 오차를 계산하여 전체 영상 시간에 맞게 오프셋(Offset)을 적용함. 중첩 구간 내의 중복 텍스트를 논리적으로 제거하여 하나의 연속된 자막으로 결합함.
* 산출물: 최종 통합 영문 자막 파일 (final_subtitle.srt).
* 확인 절차: 병합 과정 중 수정된 타임스탬프 보정값을 콘솔에 출력하고, 최종 파일 생성 후 완료 메시지 표시.

4.4. 중첩 구간 중복 제거 알고리즘 (Deduplication Logic)

* 방식: 타임스탬프 중심점 기반 절삭 (Mid-point Trimming).
* 세부 로직:
1. Part A와 Part B 사이의 중첩 구간(예: 10초) 내에서 산술적 중앙값(T_mid)을 산출함.
2. Part A 결과물에서는 시작 시간(Start Time)이 T_mid 이후인 모든 자막 블록을 제거함.
3. Part B 결과물에서는 시작 시간이 T_mid 이전인 모든 자막 블록을 제거함.
4. Part B의 모든 자막 타임스탬프에 Part A의 실제 시작 오프셋을 더하여 전체 타임라인을 재정렬함.


* 예외 처리: 중첩 구간 내에서 인식 결과가 상이할 경우, 타임스탬프의 정밀도가 더 높은(WhisperX Alignment 점수가 높은) 데이터를 우선하되 기본적으로 시간 기준 절삭을 원칙으로 함.

5. 공통 준수 사항

* 하드웨어 가용성: NVIDIA RTX 3060(VRAM 12GB) 환경을 고려하여 WhisperX 실행 시 메모리 부족(OOM)이 발생하지 않도록 적절한 배치 사이즈 및 컴퓨팅 타입(int8 또는 float16)을 적용할 것.
* 사용자 제어권: 각 모듈은 독립적으로 실행 가능해야 하며, 한 단계가 끝나면 반드시 사용자의 입력(예: Continue Y/N)을 대기하는 인터랙티브 구조로 설계할 것.
* 로그 출력: 병합 단계에서 중첩 구간 내 몇 개의 문장이 중복으로 판정되어 제거되었는지 개수를 콘솔에 실시간으로 표시할 것.
