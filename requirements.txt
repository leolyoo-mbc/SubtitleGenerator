**4. 모듈별 상세 구현 가이드라인**

**4.1. 모듈 1: 지능형 오디오 분할 (Audio Splitter - 1_split.py)**

* **기능:** 원본 m4a 파일을 ffmpeg의 스트림 카피(`-c copy`) 방식을 사용하여 1시간 단위로 분할함. 이때 n번째 파일의 종료 시점과 n+1번째 파일의 시작 시점이 10초간 겹치도록 선행 중첩 구간(Overlap)을 삽입함.
* **산출물:** 분할된 오디오 파일들(part01.m4a 등)과 각 파일의 **원본 기준 실제 시작 시간(Absolute Start Time)** 정보가 기록된 `split_metadata.json`.
* **확인 절차:** 콘솔에 분할 완료 메시지와 파일 목록을 출력하여 사용자가 파일 개수를 확인.

**4.2. 모듈 2: 텍스트 추출 (STT Extraction - 2_stt.py)**

* **모델 설정:** `faster-whisper`의 `large-v3` 모델 사용, `compute_type="float16"`, `device="cuda"`, `language="en"`.
* **기능:** 분할된 각 오디오 파일을 순차적으로 로드하여 텍스트를 추출함. VRAM 효율을 위해 파일 처리 직후 내부 캐시를 정리하거나 프로세스를 분리함.
* **산출물:** 파트별 비정렬 JSON 파일(part01_unaligned.json). WhisperX 정렬 단계에서 필요한 모든 세그먼트 데이터를 포함해야 함.
* **확인 절차:** 현재 처리 중인 파일명과 STT 진행률(Progress Bar)을 콘솔에 표시.

**4.3. 모듈 3: 강제 정렬 (Forced Alignment - 3_align.py)**

* **모델 설정:** `WhisperX`의 `align` 모듈 사용. 외부 API 호출 없이 로컬에 저장된 WAV2VEC2 모델 등을 활용하도록 설정함.
* **기능:** 2_stt.py의 JSON과 1_split.py의 오디오를 매칭하여 단어 단위(Word-level) 타임스탬프를 생성함. 각 단어의 시작(start)과 종료(end) 시간을 정밀하게 보정함.
* **산출물:** 파트별 정렬 완료 JSON(part01_aligned.json) 및 개별 검토용 SRT.
* **확인 절차:** 정렬 완료 후 개별 SRT를 통해 특정 구간의 싱크 일치 여부를 사용자가 확인.

**4.4. 모듈 4: NLP 기반 자막 병합 및 오프셋 최적화 (Sub-Merger - 4_merge.py)**

* **기능:** `split_metadata.json`의 원본 기준 시작 오프셋을 모든 타임스탬프에 합산함. 10초 중첩 구간에서 발생하는 중복 데이터를 절삭 알고리즘에 따라 제거함. 이후 `spaCy` 자연어 처리 모델을 적용하여, 단순 구두점이 아닌 문법 구조와 문맥을 분석해 의미 단위의 완벽한 한 문장으로 단어들을 묶어 최종 영문 SRT로 통합함.
* **산출물:** 최종 통합 영문 자막 파일(`final_subtitle.srt`).
* **확인 절차:** 병합된 총 문장 개수와 중복 제거된 단어 수를 로그로 출력.

**4.5. 중첩 구간 중복 제거 및 NLP 문장 재구성 알고리즘 (Deduplication Logic)**

1. **타임라인 정렬:** 각 파트의 단어 데이터에 원본 기준 절대적 오프셋을 먼저 적용하여 전체 타임라인상에 배치함.
2. **중앙값 설정:** 두 파트가 겹치는 10초 구간의 정중앙 시점(T_mid)을 결정함.
3. **단어 단위 절삭:** 이전 파트(Part A)에서는 단어의 중간값이 T_mid 이후인 단어를 제거하고, 다음 파트(Part B)에서는 단어의 중간값이 T_mid 이전인 단어를 제거함.
4. 문장 부호 복원(Punctuation Restoration) AI 모델 도입
원리: 문장 부호가 없는 날것의 텍스트를 입력받아, 오직 앞뒤 단어의 문맥과 의미 흐름만을 파악하여 문법적으로 필요한 위치에 가상의 마침표나 물음표를 스스로 예측해서 찍어주는 STT 후처리 특화 딥러닝 모델입니다.
장점: Hugging Face 등에 공개된 무료 오프라인 언어 모델(예: deepmultilingualpunctuation)을 로컬 GPU 환경에서 구동할 수 있으며, 원본 단어를 훼손하지 않으면서도 정확한 문법적 경계(Sentence Boundary)를 찾아냅니다.

**4.6. 모듈 5: 로컬 LLM 기반 문맥 번역 (Context-Aware Translator - 5_translate.py)**

* **모델 설정:** `Ollama` 로컬 환경, `gemma2:9b` 모델 사용 (Temperature=0.2).
* **기능:** 4_merge.py에서 생성된 영문 SRT를 파싱하여, 개별 자막 블록 번역 시 앞뒤 자막(Context Window 2~3개)을 프롬프트로 함께 제공함. 게임 개발 및 IT 전문 튜토리얼 강사 페르소나를 부여하여 직역을 방지하고, 전문 용어(GameObject, Component 등)를 영문으로 보존하며 자연스러운 경어체로 번역함.
* **산출물:** 최종 싱크 및 문맥이 완벽하게 맞춰진 한국어 자막 파일(`korean_subtitle_sync.srt`).
* **확인 절차:** 처리 중인 자막의 인덱스 번호와 전체 진행률을 콘솔에 실시간 표시.

**5. 공통 준수 사항**

* **VRAM 관리:** RTX 3060 환경을 고려하여 각 스크립트 종료 시 GPU 메모리가 완전히 해제되도록 설계할 것.
* **수동 연계:** 사용자가 스크립트를 하나씩 직접 실행하는 구조를 유지하여 단계별 데이터 무결성을 보장할 것.
* **에러 처리:** 경로 미존재나 파일 손상 시 즉각적으로 콘솔에 에러 원인을 명시할 것.
