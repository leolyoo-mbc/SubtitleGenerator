4.1. 모듈 1: 지능형 오디오 분할 (Audio Splitter - 1_split.py)

* 기능: 10:49:40 분량의 m4a 파일을 포맷 변환 없이(ffmpeg copy 방식) 1시간 단위로 분할하되, 이전 파트의 종료 시점보다 10초 먼저 다음 파트가 시작되도록 10초의 선행 중첩 구간(Overlap)을 삽입함. (오디오 포맷 유지를 위해 키프레임 기준으로 분할하므로 정확히 10.0초가 아닐 수 있으나, 분할 속도 및 원본 품질 유지를 우선함)
* 산출물: 공통 작업 폴더에 저장된 분할 개별 오디오 파일들 (예: part01.m4a, part02.m4a 등) 및 병합 시 누적 오차 방지를 위해 원본 파일 기준의 '절대적 분할 지점(시작/종료 타임스탬프)' 정보가 담긴 분할 메타데이터 로그 파일(JSON 또는 TXT).
* 확인 절차: 콘솔에 현재 분할 중인 파일명과 진행률을 표시하고, 작업 완료 후 폴더 내 생성된 파일 개수와 용량을 사용자가 직접 확인.

4.2. 모듈 2: 텍스트 추출 (STT Extraction - 2_stt.py)

* 모델 설정: `faster-whisper` large-v3 모델을 로드하여 분할된 오디오 파일들에 대한 1차 STT(언어는 "en"으로 고정)를 수행함. RTX 3060 12GB VRAM 환경에 맞춰 compute_type="float16"을 적용함.
* 기능: 분할된 각 오디오 파일의 음성을 텍스트로 변환하여, 타임스탬프와 텍스트가 포함된 초기 데이터(비정렬 상태)를 추출함. 다음 모듈(WhisperX)로 데이터를 온전히 넘기기 위해 중간 산출물로 저장함. 스크립트 실행이 종료되면 Python 프로세스가 종료되므로 GPU VRAM이 자동으로 안전하게 확보(OOM 방지)됨.
* 산출물: 공통 작업 폴더에 저장된 각 파트별 비정렬 JSON 파일 (예: part01_unaligned.json).
* 확인 절차: 실시간 CLI 로그를 통해 현재 처리 중인 오디오 파트와 STT 진행 상황을 출력. 작업 완료 후 사용자가 생성된 JSON 파일을 열어 텍스트 추출 누락 여부를 확인.

4.3. 모듈 3: 강제 정렬 (Forced Alignment - 3_align.py)

* 모델 설정: `WhisperX`의 정렬(align) 모듈만을 독립적으로 불러옴. Hugging Face API 토큰 입력을 요구하지 않도록 정렬용 언어 모델의 로컬 다운로드 및 캐시 사용 방식을 적용함.
* 기능: 2_stt.py에서 생성된 비정렬 JSON 파일과 1_split.py에서 생성된 개별 오디오 파일(m4a)을 쌍으로 매칭하여 불러온 뒤, WhisperX의 `.align` 기능을 사용하여 단어 단위(Word-level) 타임스탬프를 정밀하게 강제 정렬(교정)함.
* 산출물: 공통 작업 폴더에 저장된 각 파트별 정렬 완료 JSON 및 개별 검토용 SRT 파일 (예: part01_aligned.json, part01.srt). (이 JSON 파일은 WhisperX의 최종 출력 포맷인 'Segments & Words' 구조를 가짐)
* 확인 절차: 정렬 처리 진행률을 콘솔에 출력. 완료 후 사용자가 생성된 개별 SRT 파일을 재생하여 단어 단위 싱크가 정확히 맞는지 1차 검토 후 다음 병합 스크립트 실행 여부를 결정.

4.4. 모듈 4: 자막 병합 및 오프셋 최적화 (Sub-Merger - 4_merge.py)

* 기능: 1_split.py의 로그 파일(절대적 분할 지점)과 3_align.py의 결과물(정렬된 JSON)을 입력받음. 중첩 구간으로 인해 발생하는 타임스탬프 오차를 계산하여 전체 영상 시간에 맞게 오프셋(Offset)을 적용함. 단어 단위 분석을 통해 중첩 구간 내의 중복 텍스트를 정밀하게 제거하여 하나의 연속된 자막으로 결합함.
* 산출물: 공통 작업 폴더에 생성되는 최종 통합 영문 자막 파일 (final_subtitle.srt).
* 확인 절차: 병합 과정 중 수정된 타임스탬프 보정값과 중첩 구간의 절삭 내역을 콘솔에 출력하고, 최종 파일 생성 후 완료 메시지 표시.

4.5. 중첩 구간 중복 제거 알고리즘 (Deduplication Logic - 4_merge.py 내 적용)

* 방식: 단어 단위(Word-level) 타임스탬프 중심점 기반 정밀 절삭 (Mid-point Trimming) 및 문장 온전성 보존.
* 세부 로직:

1. Part A(이전 파트)와 Part B(다음 파트) 사이의 10초 중첩 구간 내에서 산술적 중앙값(T_mid) 타임스탬프를 산출함.
2. 각 JSON 파일에 기록된 개별 단어(Word)의 타임스탬프를 순회함.
3. Part A 결과물에서는 단어의 시작 시간(Start Time)이 T_mid 이후인 모든 단어 데이터를 제거함.
4. Part B 결과물에서는 단어의 시작 시간이 T_mid 이전인 모든 단어 데이터를 제거함.
5. 절삭된 단어 데이터를 바탕으로 문장(Segment)을 강제로 재구성함. 이때 T_mid 지점에서 자막이 두 개로 쪼개지는 파편화를 방지하기 위해, T_mid에 걸친 문장은 그 온전함을 유지하도록 강제 결합함(문장의 시작 타임스탬프는 Part A의 해당 문장 시작점, 끝 타임스탬프는 Part B의 해당 문장 종료점을 따름). 만약 양쪽의 STT 텍스트 인식 결과가 미세하게 다를 경우, 문장의 앞부분 문맥을 온전히 담고 있는 Part A의 텍스트를 우선적으로 채택하여 유지하는 업계 표준 방식을 따름.
6. Part B의 모든 자막 타임스탬프에 1_split.py 로그에 기록된 원본 오디오 기준의 '절대적 시작 오프셋'을 더하여 전체 타임라인을 누적 오차 없이 재정렬함.
7. 공통 준수 사항

* 하드웨어 가용성: NVIDIA RTX 3060(VRAM 12GB) 환경에서 메모리 부족(OOM)이 발생하지 않도록 텍스트 추출(모듈 2)과 강제 정렬(모듈 3)을 물리적으로 완벽히 분리하였으며, 각 모듈 내에서도 적절한 배치 사이즈 및 컴퓨팅 타입(float16)을 적용할 것.
* 사용자 제어권: 각 작업은 `1_split.py`, `2_stt.py`, `3_align.py`, `4_merge.py` 형태의 완전 독립된 마이크로 스크립트로 설계되어, 사용자가 각 단계별 산출물을 직접 확인한 뒤 다음 단계를 실행하는 구조로 작성할 것.
* 에러 핸들링 및 로그: 각 스크립트는 입력 파일이 공통 작업 폴더에 존재하지 않을 경우 명확한 에러 메시지를 띄우고 종료되어야 하며, 병합 단계(모듈 4)에서는 중첩 구간 내 정리된 단어 및 문장의 병합 상태를 콘솔에 실시간으로 표시할 것.
