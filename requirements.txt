4.1. 모듈 1: 지능형 오디오 분할 (Audio Splitter - 1_split.py)

* 기능: 10:49:40 분량의 m4a 파일을 포맷 변환 없이(ffmpeg copy 방식) 1시간 단위로 분할하되, 이전 파트의 종료 시점보다 10초 먼저 다음 파트가 시작되도록 10초의 선행 중첩 구간(Overlap)을 삽입함. (오디오 포맷 유지를 위해 키프레임 기준으로 분할하므로 정확히 10.0초가 아닐 수 있으나, 분할 속도 및 원본 품질 유지를 우선함)
* 산출물: 공통 작업 폴더에 저장된 분할 개별 오디오 파일들 (예: part01.m4a, part02.m4a 등) 및 실제 분할 지점(시작/종료 타임스탬프) 정보가 담긴 로그 파일.
* 확인 절차: 콘솔에 현재 분할 중인 파일명과 진행률을 표시하고, 작업 완료 후 폴더 내 생성된 파일 개수와 용량을 사용자가 직접 확인.

4.2. 모듈 2: 텍스트 추출 및 강제 정렬 (STT & Alignment - 2_stt.py)

* 모델 설정: Hugging Face 토큰을 요구하지 않도록 `faster-whisper` large-v3 모델을 독립적으로 불러와 1차 STT(언어는 "en"으로 고정)를 수행함. 이후 추출된 텍스트 결과물만 `WhisperX` 정렬 모듈의 `.align` 기능으로 넘겨 단어 단위 강제 정렬(Forced Alignment)을 수행함.
* 기능: 분할된 각 오디오 파일에 대해 JSON 형태의 워드 단위 타임스탬프를 추출하고, 강제 정렬을 통해 싱크를 정밀하게 교정함.
* 산출물: 공통 작업 폴더에 저장된 각 파트별 JSON 및 SRT 파일.
* 확인 절차: 실시간 CLI 로그를 통해 현재 처리 중인 문장과 배치(Batch) 진행 상황을 출력. 각 파일 작업 완료 시 사용자가 JSON/SRT 파일을 열어 인식률을 검토한 뒤 다음 스크립트 실행 여부를 결정.

4.3. 모듈 3: 자막 병합 및 오프셋 최적화 (Sub-Merger - 3_merge.py)

* 기능: 중첩 구간으로 인해 발생하는 타임스탬프 오차를 계산하여 전체 영상 시간에 맞게 오프셋(Offset)을 적용함. 단어 단위(Word-level) 분석을 통해 중첩 구간 내의 중복 텍스트를 정밀하게 제거하여 하나의 연속된 자막으로 결합함.
* 산출물: 공통 작업 폴더에 생성되는 최종 통합 영문 자막 파일 (final_subtitle.srt).
* 확인 절차: 병합 과정 중 수정된 타임스탬프 보정값을 콘솔에 출력하고, 최종 파일 생성 후 완료 메시지 표시.

4.4. 중첩 구간 중복 제거 알고리즘 (Deduplication Logic)

* 방식: 단어 단위(Word-level) 타임스탬프 중심점 기반 정밀 절삭 (Mid-point Trimming) 및 문장 온전성 보존.
* 세부 로직:

1. Part A와 Part B 사이의 실제 중첩 구간 내에서 산술적 중앙값(T_mid)을 산출함.
2. 각 JSON 파일에 기록된 개별 단어(Word)의 타임스탬프를 순회함.
3. Part A 결과물에서는 단어의 시작 시간(Start Time)이 T_mid 이후인 모든 단어 데이터를 제거함.
4. Part B 결과물에서는 단어의 시작 시간이 T_mid 이전인 모든 단어 데이터를 제거함.
5. 절삭된 단어 데이터를 바탕으로 문장(Segment)을 강제로 재구성함. 이때 T_mid 지점에서 자막이 두 개로 쪼개지는 파편화를 방지하기 위해, T_mid에 걸친 문장은 그 온전함을 유지하도록 강제 결합함(문장의 시작 타임스탬프는 Part A의 해당 문장 시작점, 끝 타임스탬프는 Part B의 해당 문장 종료점을 따름).
6. Part B의 모든 자막 타임스탬프에 Part A의 실제 시작 오프셋을 더하여 전체 타임라인을 재정렬함.
7. 공통 준수 사항
